# 多智能体讨论（Debate V2）— 模型等待计时

> 全项目接口与数据流总约束见 **`docs/接口与数据流架构约定.md`**。  
> **异步 Job + SSE** 的单一实施依据为 **`docs/辩论与代码生成异步任务设计.md`**（含事件类型、降级与 §12 例外登记）。  
> 本文档当前约束「同步 RPC 阶段」的计时与 UI；待上述设计落地后，须在本文件增加「Job 模式下计时与 UI」一节，并与该设计文档交叉引用。

## 目的

在 **同步 Connect RPC** 等待大模型返回期间，向用户展示 **已等待时长**，降低「无反馈」焦虑，并与后端 `timeout_seconds` / 网关超时等设定形成心理对齐。

## 约定

| 项目 | 说明 |
|------|------|
| **计时起点** | **`PrepareDebateV2ChatJob`**（至 SSE+Run 完成）、**`RunDebateV2ChatJob`**、**`StartDebateV2AdvanceJob`**、**`StartDebateV2RejectCodeJob`**（Unary 提交后直至对应 SSE Job 完成）、**`AdvanceDebateV2`**（不经 Job 的短步进）；保留的 Unary **`ChatDebateV2`** / **`RejectDebateV2Code`** 若被调用亦计时。 |
| **不计时** | `StartDebateV2`（仅建会话）、`BackDebateV2`（无模型）不启动计时。 |
| **显示粒度** | 秒级递增，展示格式为 `M:SS`（例如 `0:05`、`1:32`）。 |
| **状态来源** | 前端 `useDebateFlow` 内单一状态机：`modelWaitStartedAt` + `modelWaitElapsedSeconds`，`useEffect` 每秒 tick；请求 `finally` 中清零。 |
| **展示位置** | 对话步：消息列表下方 **Alert** + 加载气泡旁文案；代码步：沿用 Spin 区域下的 **已等待** 行，与对话步同一套秒数。 |
| **步骤条与主区（乐观）** | `Advance` 为同步 RPC，服务端在返回前才会更新 `session.currentStep`。点击「下一步」后，前端根据 `stepLabels` 立即设置 **乐观步骤**（与后端 `nextStepKey` 一致），无专家时从意图直接进入 **代码** 时，主区会马上切到代码生成 Spin，避免长时间仍显示「澄清意图」。RPC 结束或失败后在 `finally` 中清除乐观状态。 |
| **文案与 i18n** | 使用 `ai.debate.v2.modelWaitBanner`、`modelWaitBubble`、`codeElapsed`（见各语言 `ai_debate.ts`）。 |

## 实现位置

- 逻辑：`frontend/src/pages/ai/debate/flow/useDebateFlow.ts`
- UI：`frontend/src/pages/ai/debate/DebatePageV2Steps.tsx`、`DebatePageV2.tsx`
- 文案：`frontend/src/i18n/resources/*/ai_debate.ts`

## 后续可选

- 若接入流式输出，可将计时与 **首 token 时间** 并列展示。
- 若网关/代理断开流（如 `SubscribeEvents`），与本计时无关；见运维侧 HTTP/2 与超时配置。
