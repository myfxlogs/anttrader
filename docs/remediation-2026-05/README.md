# AntTrader 项目整改文档入口（2026-05）

本目录是 AntTrader 下一阶段整改的**新权威文档入口**。

旧 `docs/` 下与当前代码和逻辑不符的历史文档不再保留为阅读入口。新开发、新重构、新评审应优先以本目录为准。

## 1. 整改核心目标

本轮整改不以新增功能为主，而以系统治理、架构收敛和交易链路稳定为主：

1. **前后端职责重新划线**
   - 前端只负责展示、交互、渲染、表单输入、轻量本地 UI 状态。
   - 所有业务计算、统计计算、风控计算、交易决策、指标聚合、AI 调度、回测与策略执行全部放在后端或策略服务。

2. **通信协议统一回归 ConnectRPC**
   - 浏览器到后端的业务通信统一使用 ConnectRPC。
   - 禁止新增业务 REST 接口。
   - 现有 `/api/...` 业务 REST 端点纳入迁移清单。
   - `/health` 等非业务探针可作为少量例外保留，但不得承载业务逻辑。

3. **代码规模硬约束**
   - 除 Markdown 文档外，任何代码文件、脚本文件、配置生成代码、proto 生成代码单文件不得超过 800 行。
   - 超过 800 行必须拆分，不允许以“自动生成”为理由例外。

4. **文档体系重建**
   - 建立新的整改文档目录。
   - 旧文档只作为历史参考，逐步合并到新文档体系。
   - 文档必须区分“当前实现”“目标设计”“迁移计划”“废弃内容”。

## 3. 执行原则

整改顺序必须遵守：

```text
先文档定边界
  → 再清点现状
  → 再迁移接口
  → 再拆分大文件
  → 再重构核心链路
  → 最后删除旧代码和旧文档
```

任何破坏性删除都必须先确认已有当前权威替代，且删除后不会把历史计划误读为当前实现。
