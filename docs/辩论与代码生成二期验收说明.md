# 辩论与代码生成 — 二期验收说明

> 与 **`docs/辩论与代码生成异步任务设计.md`** 配套；用于手工回归与发布前核对。  
> 自动化：`backend` 执行 `go test ./...`；`frontend` 执行 `npm run build` 与 `npm run lint`。

---

## 1. 环境

| 项 | 要求 |
|----|------|
| 后端 | 已注册 `GET /antrader/sse/debate-v2/advance-jobs/{jobId}/stream` 与 `.../chat-jobs/{jobId}/stream`；用户 JWT 可经 Query `access_token` 访问 SSE。 |
| 反代 | 与 **`docs/边缘网关与长连接问题处理参考.md`** 一致：`proxy_buffering off`、足够 `proxy_read_timeout`；`/antrader` 含上述 SSE 路径。 |
| 单副本 | 辩论 Job 为**进程内** hub；多副本 `backend` 会导致 SSE 与 Worker 不一致，验收时 **只跑一个 backend 副本**。 |

## 2. 手工用例（浏览器）

### 2.1 Chat（澄清意图）

1. 新建辩论 V2 会话，停留在 **澄清意图**。  
2. 发送一条用户消息。  
3. **Network**：存在 **`PrepareDebateV2ChatJob`** 与 **`RunDebateV2ChatJob`** Unary；**Prepare** 后立刻对 **`.../chat-jobs/<jobId>/stream`** 建立 **EventSource**（`text/event-stream`），再 **Run**。  
4. UI：助手内容随 **`chunk`** 递增（或模型一次性吐字时整段到达）；结束后会话刷新，无长时间 Unary 挂起。

### 2.2 Advance（进入专家 / 代码）

1. 从意图 **下一步** 进入 agent 或代码步（视是否选专家）。  
2. **Network**：**`StartDebateV2AdvanceJob`** + **`.../advance-jobs/<jobId>/stream`**。  
3. 代码步：流式区与最终 `GetDebateV2Session` 一致。

### 2.3 拒绝并重写代码（Reject Job）

1. 会话在 **代码** 步且已有生成结果。  
2. 提交 **拒绝反馈**（触发重写）。  
3. **Network**：**`StartDebateV2RejectCodeJob`** Unary；SSE 仍为 **`advance-jobs`**（与 advance 共用）；**不应**出现对 `chat-jobs` 的新订阅（除非另发聊天）。  
4. UI：重写过程有流式预览；完成后代码区更新。

### 2.4 SSE 断线行为（可选）

1. DevTools → Network → 对 `stream` 请求选 **Offline** 再恢复，或短暂停后端再启。  
2. 预期：前端 **退避重连** 同一 SSE URL（非高频 `Get*Job` 轮询）；极端失败后出现可理解错误提示，且可 **单次** `Get*Job` 对齐终态。

## 3. 兼容性 Unary（非主路径）

| RPC | 用途 |
|-----|------|
| `ChatDebateV2` | 旧客户端或调试整段聊天。 |
| `RejectDebateV2Code` | 旧客户端整段拒绝重写。 |
| `AdvanceDebateV2` | 无 LLM 的步进。 |

主路径 UI 应使用 **Start*Job + EventSource**。
