# 技术债清单与治理状态

本文在仓库范围内做一次**静态与依赖面**扫描的结论，用于排期治理；**数字为数量级**（随分支变动），以 `grep` / 人工复核为准。

## 1. 已对齐或近期已治理项

| 项 | 说明 |
|----|------|
| Protobuf-ES / Connect 浏览器栈 | `frontend` 使用 `@bufbuild/protobuf` **v2**、`@connectrpc/connect` **v2**、`protoc-gen-es` **v2**；`frontend/src/gen` 生成头均为 **protoc-gen-es v2**，无 v1 `extends Message` 混存。约定见 **`.cursor/rules/protobuf-es-v2.mdc`**。 |
| `tools/proto-gen` | 仅保留 **`@bufbuild/buf`** 依赖；已清除锁文件中遗留的 **v1 protobuf / 废弃 connect-web 插件** 记录。 |
| `make proto` 与 `buf generate` | Makefile 已与 `frontend/package.json` 的 `gen:proto` 一致，统一为仓库根执行 **`buf generate`**（不再带易混淆的 `proto` 尾参）。 |
| `protoc-gen-connect-go` | **v1.19.1** 要求 **Go ≥ 1.24**；Makefile `proto-tools` 已注释说明；CI 使用 **Go 1.25.6**。 |
| GitHub Actions 前端 | **Node 22** 与本地/容器常用版本对齐，降低与 Vite 7 生态差异。 |
| `frontend/Dockerfile` 构建镜像 | 默认 **`hub.rat.dev/library/node:22-alpine`**，与 CI 一致；可通过 **`NODE_BUILD_IMAGE`**（`docker-compose.yml` 的 `build.args` 或 `docker build --build-arg`）覆盖，例如改为 `docker.io/library/node:22-alpine`。 |
| CI **proto-drift** job | **`make proto-tools && make proto`** 后若 **`git status`** 非空则失败，防止 `.proto` 已改但未提交生成物。 |

## 2. 中优先级技术债（建议分期排期）

### 2.0 本轮复扫摘要（2026-05-03）

- **扫描口径**：排除 `node_modules`、`.git`、`dist`、`gen`、`__pycache__`、`.pytest_cache` 后，复扫依赖清单、弃用/旧 API 信号、`TODO/FIXME/HACK/XXX`、类型检查禁用/忽略、手写 `any/as any` 与超 800 行代码文件。
- **标记注释**：手写区约 **8** 处，主要是 AI prompt / i18n 文案中的示例 `TODO`，不是待实现代码；可安全保留。
- **类型检查禁用/忽略**：约 **20** 处、**15** 个文件；前端集中在 `systemai/hooks.ts`、策略启动表单等，后端策略服务多为测试或 Python 版本兼容场景。
- **前端 `any/as any`**：手写 `frontend/src` 约 **518** 处、**68** 个文件；热点为 `StrategySchedulePage.tsx`、`StrategyScheduleLogsPage.tsx`、`StrategyTemplatePage.tsx`、`ScheduleTable.tsx`、`LogManagement.tsx`、账户页与 `pythonStrategy`/`trading` client。
- **弃用/旧 API 信号**：约 **343** 处，主要来自 `backend/mt4/*.pb.go`、`backend/mt5/*.pb.go` 生成物中的 Go protobuf `Descriptor()` deprecated 注释；不应手改生成物。
- **依赖面**：前端 Connect / Protobuf-ES 仍为 v2；`tools/proto-gen` 仅保留 `@bufbuild/buf`；后端 `github.com/lib/pq` 与 `pgx/sqlx/gorm` 并存；策略服务 `requirements.txt` 依赖整体固定，但 FastAPI / Uvicorn / Pydantic / pandas 等版本偏旧，升级需回归。
- **超 800 行代码文件**：`backend/mt4/mt4.pb.go`、`backend/mt4/mt4_grpc.pb.go`、`backend/mt5/mt5.pb.go`、`backend/mt5/mt5_grpc.pb.go` 为生成物，仍违反当前“代码文件不超 800 行”规则，建议通过生成拆包或 wrapper 分层处理；`frontend/package-lock.json` 为锁文件大文件，属依赖治理项。

### 2.0.1 可安全修复项（小 PR）

- **补充 ESLint/TS 排除口径**：确保 `frontend/dist`、`frontend/src/gen` 不参与手写债务统计；只调整工具配置或文档，不影响业务。
- **前端局部 `unknown` 收窄**：优先处理 `catch (e: any)`、事件回调参数、`window/navigator` 扩展访问等低风险点；每次按单页或单 client 控制 diff。
- **测试内 `type: ignore` 注明原因**：`strategy-service/tests` 中为验证 frozen 或故意传错类型的忽略，可保留但建议统一写法与最小作用域。
- **文案型 TODO 标注白名单**：AI prompt / i18n 里的 `# TODO:` 属输出模板，占位说明可加入扫描白名单，避免后续误报。
- **依赖镜像策略文档化**：锁文件 registry 混用在完整性哈希保护下可运行，短期可只统一 `.npmrc` 约定；重写 lock 应单独 PR。

### 2.0.2 需分期处理项（中/大 PR）

- **策略页类型收敛**：`StrategySchedulePage.tsx`、`StrategyScheduleLogsPage.tsx`、`StrategyTemplatePage.tsx`、`ScheduleTable.tsx` 等与生成类型、表格列、表单值耦合，建议按“client 返回类型 → hooks → 页面组件”分层推进。
- **交易/账户 client 类型化**：`frontend/src/client/trading.ts` 仍从空/聚合 `api_pb` 导出 `Order`，且多处 `response: any`；应先确认实际生成消息文件与 Connect 返回类型，再替换页面依赖。
- **Python 策略 client 类型化**：`frontend/src/client/pythonStrategy.ts` 同时承担旧接口兼容、BacktestRun、流式订阅，需结合生成 schema 与页面模型拆分，避免一次性改动影响回测链路。
- **MT4/MT5 protobuf 大文件治理**：当前生成物显著超过 800 行且含大量 deprecated 注释；不能手改，应从 `.proto` 拆包、生成策略或适配层隔离入手，并配套桥接端回归。
- **后端数据库驱动统一**：`lib/pq` 仍在回测/仓储路径使用，迁到 `pgx` 需验证数组、时间、事务、COPY 或扫描行为，属于行为等价迁移。
- **策略服务依赖升级**：FastAPI、Uvicorn、Pydantic、pandas 等固定版本升级可能影响 API 序列化、数据处理与沙箱执行；需建立测试基线后分批升级。

### 2.1 前端类型与 `any`

- **`frontend/src/client/ai.ts`**：已对齐 **protobuf-es v2**（`create`/`timestampDate`、生成消息类型），去掉 **`toAgentView(a: any)`** 与 **`(aiClient as any)`** 等弱类型写法。
- **`frontend/src/client/strategy.ts`**：RPC 响应改为 **Connect 推断类型**；`RunBacktestResult.metrics` 使用 **`BacktestMetrics`**；`executeSignal` 的 **`executedAt`** 使用 **`timestampDate`**；模板参数与 **`ScheduleConfig`** 使用 **`PartialMessage<>`** / **`create(ScheduleConfigSchema, …)`**；类型再导出改为自 **`strategy_*_pb` / `common_pb`**（不再从空的 **`api_pb`** 再导出）。
- **`frontend/src/client/log.ts`**：去掉 **`response: any`** 与 **`logClient as any`**；调度运行日志返回 **`ScheduleRunLog[]`**；与 **`@/types/log`** 不完全一致的页面已改为使用 **`@/gen/log_*_pb`** 中的生成类型（如 **`ConnectionLog`**）。
- **`frontend/src/client/stream.ts`**：回调与共享流使用 **`OrderUpdate` / `ProfitUpdate` / `AccountStatusEvent` / `StreamEvent`** 等；去掉 **`as any`** 迭代；**`subscribeUserSummary`** 回调类型与 **`UserSummary`** 对齐。
- **`frontend/src/providers/ConnectProvider.tsx`**：订单 / 盈亏 / 状态流与 **`Position` / `OrderUpdate` / `ProfitUpdate` / `AccountStatusEvent`** 对齐，去掉大面积 **`as any`**。
- **`frontend/src/pages/ai/debate/*`**：**`useDebateFlow`** 占位 agent 与 **`DebatePageV2Steps`** 保存模板路径已去掉 **`any`**。
- **仍待收紧**：**`StrategyScheduleLogsPage.tsx`**、**`StrategySchedulePage.tsx`**、**`StrategyTemplatePage.tsx`**、**`Dashboard.tsx`**、**`SystemAI.tsx`**、**`frontend/src/pages/accounts/`** 下各组件、**`frontend/src/client/trading.ts`** 等仍有 **`any` / `as any`**，可按页面与 **`trading`/`account` client** 分批改为生成类型或 **`unknown` + 窄化**。
- **`frontend/src/stores/aiStore.ts`**：发送消息路径的 **`catch (e: any)`** 已改为 **`unknown`**（与 `toFriendlyAIChatError` 对齐）。
- **治理方式**：按模块逐步替换为 **生成 DTO 类型** 或 **Zod/手写窄类型**；不必单次大 PR。

### 2.2 生成代码 ESLint 全禁用

- **`frontend/src/gen/*_pb.ts`**：生成器自带 `/* eslint-disable */`，属工具链行为；**不要手改生成文件**。若需更严规则，应在 **buf / protoc-gen-es 选项** 或 **eslint overrides** 中针对 `src/gen` 单独配置（与手写代码区分）。

### 2.3 ~~CI 未校验「生成物与 proto 一致」~~（已加 job）

- 已在 **`.github/workflows/ci.yml`** 增加 **`proto-drift`** job：`npm ci`（`tools/proto-gen` + `frontend`）→ **`make proto-tools && make proto`** → **`git status --porcelain`** 必须为空。

### 2.4 `npm` 锁文件镜像混用

- **`frontend/package-lock.json`** 中大量包解析到 **`registry.npmmirror.com`**，部分为 **`registry.npmjs.org`**，与 CI/海外环境行为可能略有差异（**完整性哈希**仍以 lock 为准）。
- **治理方式**：团队统一 **`.npmrc` registry** 后 **`npm install` 重写 lock**（一次性大 diff）；或接受现状仅文档说明。

## 3. 低优先级 / 架构级债

### 3.1 后端 `lib/pq` 与 `pgx` 并存

- **`github.com/lib/pq`** 仅见于 **回测相关** repository/service（如 `backtest_run_repository.go`），主路径已 **pgx/sqlx**。
- **治理**：若需统一栈，可评估将 **数组类型 / COPY** 等 pq 专用逻辑迁到 **pgx**，再移除 `lib/pq`；属**行为等价**迁移，需单测与回归。

### 3.2 文档中的「TODO」字样

- **i18n 提示词模板**、**辩论 prompt** 中含「`# TODO:`」等，为**给模型/用户的占位说明**，不是代码未完成标记；**不应**当技术债删除，除非产品文案要求改表述。

### 3.3 历史治理与 REST 存量

- **`docs/remediation-2026-05/`** 下多篇迁移与清单文档描述 **REST → ConnectRPC** 的进程；当前仍以 **`docs/接口与数据流架构约定.md`** 为接口形态纲领。
- **REST 业务面**是否清零以该批文档 +代码为准，本清单不重复枚举路径。
