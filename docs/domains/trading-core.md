# 交易核心边界

## 1. 职责范围

交易核心负责账户交易权限、风控校验、订单执行、执行日志、订单日志与审计日志的后端权威记录。

## 2. 后端边界

- `TradingService` 处理手动交易入口和 MT 执行调用。
- `AutoTradingService` 处理自动交易信号执行与自动交易风控。
- `RiskEngine` 提供手动、自动、调度统一风控决策。
- `ExecutionGateway` 统一记录执行过程、幂等、熔断、风险拒绝和引擎错误。

## 3. 日志关系

- 执行日志记录策略或自动交易的一次执行生命周期。
- 订单日志记录 MT 订单返回后的交易事实。
- 审计/操作日志记录用户或系统触发的操作、风险决策和错误原因。
- 前端不得通过本地规则推导日志关系，只展示后端返回字段。

## 4. MT4/MT5 adapter 边界

- 平台差异收敛在连接管理和执行 adapter 内。
- 交易服务只依赖统一的 `MatchingEngine`/订单请求响应模型。
- 禁止在页面或 Connect handler 中分散 MT4/MT5 平台判断。
- 外部 MT 协议文件按上游合同处理，不作为业务 proto 拆分对象。

## 5. 风控兼容策略

- 默认兼容既有放行路径。
- 明确危险输入由 `RiskDecision` 拒绝。
- 账户禁用、投资者账户、非法 symbol、volume、position、balance、equity 均由后端校验。
